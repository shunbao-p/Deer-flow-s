# Weather Skill

## 概述

这是一个简单的天气查询技能，当用户询问"天气"或"天气预报"时自动触发。该技能使用web_search工具搜索网络上的天气信息，提取并返回当前温度。

## 使用场景

当用户提出以下类型的问题时触发此技能：
- "天气怎么样？"
- "今天天气如何？"
- "天气预报"
- "现在温度多少？"
- "本地天气"

## 工作流程

1. **触发检测**：当用户查询中包含"天气"或"天气预报"关键词时触发技能
2. **位置识别**：从用户查询中提取城市/位置信息（如果用户未指定，默认查询本地天气）
3. **网络搜索**：使用web_search工具搜索"[城市名] 当前天气"或"weather in [city]"
4. **信息提取**：从搜索结果中提取当前温度信息
5. **结果呈现**：向用户返回温度信息，包括摄氏度（°C）和华氏度（°F）转换

## 核心能力

### 1. 智能触发
- 关键词匹配：检测"天气"、"天气预报"、"温度"等关键词
- 中文支持：完全支持中文查询
- 位置感知：可以识别用户指定的城市或地区

### 2. 网络搜索策略
- 搜索查询构建：自动生成优化的天气搜索查询
- 多源信息：从多个搜索结果中提取最相关的温度数据
- 实时数据：获取最新的天气信息

### 3. 温度提取
- 数据解析：从搜索结果中识别温度数值
- 格式转换：提供摄氏度和华氏度显示
- 单位处理：正确处理°C和°F单位

## 使用方法

当技能被触发时，将自动执行以下操作：

```python
# 示例：用户查询"北京天气"
# 1. 检测到"天气"关键词，触发技能
# 2. 提取城市"北京"
# 3. 搜索"北京 当前天气"
# 4. 从搜索结果提取温度
# 5. 返回："北京当前温度：25°C (77°F)"
```

技能可以通过以下方式进行配置：

### 温度单位偏好
- 默认显示摄氏度和华氏度
- 可根据用户偏好调整

### 位置处理
- 自动检测用户提到的城市

## 限制和注意事项

1. **数据准确性**：依赖网络搜索结果的准确性
2. **网络依赖**：需要互联网连接
3. **位置精度**：可能需要用户明确指定城市以获得准确结果
4. **语言支持**：主要支持中文，也支持英文天气查询
